# Leave System Overhaul (ระบบตั้งค่าประเภทวันลาแบบ Dynamic)

## 📌 สรุปการเปลี่ยนแปลง
ตามความต้องการใน [SPEC_ระบบตั้งค่าประเภทวันลา.md](../SPEC_ระบบตั้งค่าประเภทวันลา.md) เราได้ทำการปรับปรุงโครงสร้างจากเดิมที่ฮาร์ดโค้ดประเภทวันลาไว้ในแอป ให้กลายเป็น **"ระบบตั้งค่าแบบยืดหยุ่น (Dynamic)"** ซึ่งผู้ใช้สามารถ:
1. **เพิ่ม/แก้ไขประเภทวันลาเองได้** ผ่านเมนู "⚙️ ตั้งค่าวันหยุด" (Leave Settings Overlay)
2. **ปรับแต่งสิทธิประโยชน์ของแต่ละประเภทได้อิสระ**:
   - การรับค่าจ้าง (ได้เงิน/ไม่ได้เงิน/ลดทอน)
   - การหักเบี้ยขยัน (หัก/ไม่หัก)
   - หน่วยนับ (วัน/ชั่วโมง)
   - การจัดการโควตา (มีโควตาและรีเซ็ตรายปี/รายเดือน)
3. **กำหนดสีประจำประเภท (Tag Color)** และซ่อน/แสดงในหน้าเพิ่มรายการได้

## 🛠️ ไฟล์ที่แก้ไข
1. **`js/utils.js`**
   - เพิ่ม `DEFAULT_LEAVE_TYPES` (ประเภทวันลามาตรฐาน 21 ประเภท ครอบคลุมกฎหมายแรงงานไทย)
   - เพิ่มฟังก์ชัน `getLeaveTypes()` และ `saveLeaveTypes()` สำหรับเรียกใช้ข้อมูลและบันทึกลง `localStorage` (Merge กฎใหม่เข้ากับข้อมูลผู้ใช้เดิม)
2. **`index.html`**
   - ถอด `<select id="leaveType">` ที่ถูกล็อกไว้ออก เพื่อรอรับข้อมูล Dynamic จาก JS
   - เพิ่มปุ่ม "🗂️ ตั้งค่าประเภทวันหยุด & โควตา" ลงในหน้าตั้งค่าหลัก
   - สร้าง Overlay สำหรับจัดการรายการ (Layer 2) และฟอร์มแก้ไขโควตา/หน่วย (Layer 3)
3. **`js/calc.js`**
   - รีแฟคเตอร์ `getBanks()` ให้ดึงยอดโควตาตั้งต้นจาก `quota_total` แบบ Dynamic แทน
   - รีแฟคเตอร์ `periodStats()` ให้รองรับการนับสถิติวันลาทั้งหมด และตัดค่าใช้จ่าย/เบี้ยขยัน ตามเงื่อนไขใน Config (is_paid, docks_diligence, wage_deduction) แบบยืดหยุ่น
4. **`js/settings.js`**
   - เพิ่มฟังก์ชันวาด UI วันลาในตั้งค่า (`renderLeaveSettings`, `openEditLeaveType`)
   - แก้ไข `openEntryModal` ให้สร้างตัวเลือกประเภทลาจากการตั้งค่าใหม่เสมอ
   - รองรับ "Reverse-Calculate" สำหรับปรับโควตาคงเหลือปัจจุบัน 
5. **`tests/calc-tests.html`**
   - อัปเดตชุดทดสอบให้โหลด `utils.js` ก่อนเรียก `calc.js` เพื่อไม่ให้เกิด Error `getLeaveTypes is not defined`

## ✅ สิ่งที่รองรับการใช้งานแล้ว
- รองรับประเภทวันลา 21 ประเภท (กลุ่ม A: ตามกม. 15 ข้อ และกลุ่ม B: บริษัท 6 ข้อ)
- **การหยุดผลิต (Production Stop):** เตรียมโครงสร้างหักค่าจ้าง 25% (จ่าย 75%) ไว้แล้วใน Config (`wage_deduction: 0.25`)
- ประวัติข้อมูลเดิม (Legacy data) ยังคงถูกดึงขึ้นมาคำนวณได้อย่างถูกต้องผ่าน fallback logic

## ⚠️ การตั้งค่าเพิ่มเติมของแต่ละบริษัท
ประเภทการลาอย่างเช่น **ลาป่วย (Sick Leave)** ห้ามหักเบี้ยขยันหรือไม่? ตามกม.สามารถนำมาเป็นเงื่อนไขตัดเบี้ยขยันได้ (ส่วนใหญ่บริษัททำกัน) แต่ลากิจพิเศษบางประเภท หากอิงนโยบายบริษัทเป็นหลัก สามารถปรับการตั้งค่า "หัก/ไม่หักเบี้ยขยัน" ในแอปตามกฎหมายบริษัทของคุณได้อย่างอิสระ
