# Walkthrough: การตั้งค่าปุ่ม Enter (UX/UI Keyboard Behavior)

## 📌 สรุปการเปลี่ยนแปลง (Changelog)

เพื่อปรับปรุงประสบการณ์ผู้ใช้งาน (UX) บนอุปกรณ์มือถือ (iOS/Android) เวลาที่ผู้ใช้กรอกข้อมูลในหน้าจอการตั้งค่า (Settings) และกดปุ่ม **Enter** (หรือ Next/Done) บนคีย์บอร์ด ได้ทำการปรับพฤติกรรมของแอปดังนี้:

### 1. ป้องกันการกระโดดไปยังปุ่มบันทึก (Prevent Auto-scroll)
- **ปัญหาที่พบ:** เวลาผู้ใช้กด Enter ในฟิลด์แบบเดี่ยวๆ หน้าจอจะเลื่อนลงไปที่ปุ่ม "บันทึก" ด้านล่างสุด หรือทำการ Submit ทันที ซึ่งทำให้รู้สึกเหมือนโดนระบบ "เร่ง" ทั้งที่ยังกรอกข้อมูลส่วนอื่นไม่เสร็จ
- **การแก้ไข:** ดักจับ Event ของคีย์บอร์ด (`keydown` สำหรับปุ่ม `Enter`) บนช่อง input ทุกช่องเพื่อป้องกันพฤติกรรมนี้

### 2. กำหนดพฤติกรรมของปุ่ม Enter อย่างเหมาะสมตามบริบท (Contextual Enter Key)
- **สำหรับ Input เดี่ยวๆ (เช่น ช่องหักอื่นๆ, ภาษี, ประกันสังคม):** เมื่อกด Enter ระบบจะทำการพับคีย์บอร์ดลง (Blur) เพื่อให้ผู้ใช้ตรวจทานข้อมูลที่เพิ่งกรอกไป แทนที่จะพาเลื่อนไปล่างสุด
- **สำหรับ Input ที่มีความเกี่ยวข้องกัน (ฟอร์มจัดการวันหยุด):** ได้เขียน Flow พิเศษไว้เพื่อความลื่นไหล:
  - ช่อง **"วันที่เริ่ม"** -> กด Enter แล้วจะย้าย Focus ไปที่ช่อง **"ถึงวันที่"**
  - ช่อง **"ถึงวันที่"** -> กด Enter แล้วจะย้าย Focus ไปที่ช่อง **"ชื่อวันหยุด"**
  - ช่อง **"ชื่อวันหยุด"** -> กด Enter แล้วจะทำการพับคีย์บอร์ดลง

### 3. เพิ่มความสวยงามและ Native UX (Keyboard Hint)
- เพิ่ม Attribute `enterkeyhint="next"` และ `enterkeyhint="done"` ในโครงสร้าง HTML หน้าจัดการวันหยุด 
- เพื่อให้คีย์บอร์ดของมือถือ (เช่น iOS Safari) แสดงคำใบ้บนปุ่ม Enter ได้อย่างถูกต้อง (เช่น แสดงคำว่า "ถัดไป" เมื่อต้องกรอกฟิลด์ต่อไป และคำว่า "เสร็จสิ้น" ในช่องสุดท้าย)

## 📁 ไฟล์ที่เกี่ยวข้องกับการแก้ไข
1. `index.html`: เพิ่มแอตทริบิวต์ `enterkeyhint` ให้ช่อง Input
2. `js/app.js`: เพิ่ม Logic ควบคุม `document.addEventListener('keydown', ...)` สำหรับปุ่ม Enter
