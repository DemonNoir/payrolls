# Handoff Skill

เมื่อผู้ใช้พิมพ์คำสั่ง `/handoff` หรือขอให้ "ส่งต่องาน" ให้คุณทำตามขั้นตอนต่อไปนี้ทันที:

1. **รวบรวมข้อมูลบริบทปัจจุบัน:** วิเคราะห์และรวบรวมสิ่งที่เกิดขึ้นในเซสชันนี้ เช่น งานที่เพิ่งทำเสร็จ, ไฟล์ที่มีการแก้ไข, โค้ดที่ยังทำค้างอยู่, บั๊กที่ยังแก้ไม่สำเร็จ หรือไอเดียที่ต้องทำในอนาคต
2. **สร้างหรืออัปเดตไฟล์ Handoff:** ใช้เครื่องมือเพื่อเขียนสร้างหรืออัปเดตไฟล์ `HANDOFF.md` ใน root directory ของโปรเจกต์
3. **โครงสร้างของไฟล์ HANDOFF.md:** ไฟล์ควรมีรูปแบบ Markdown โดยครอบคลุมหัวข้อหลักดังนี้:
   - **Project Overview:** ภาพรวมของโปรเจกต์ (สั้นๆ)
   - **Current State:** สถานะล่าสุดของงาน และเซสชันการทำงานล่าสุด
   - **Recent Changes:** การเปลี่ยนแปลงที่เพิ่งทำในเซสชันนี้ (ระบุชื่อไฟล์ที่แก้ไข)
   - **Pending Tasks / Next Steps:** งานที่ยังค้างอยู่ หรือสิ่งที่ผู้ใช้ต้องการให้ทำต่อในเซสชันหน้า
   - **Open Issues / Notes:** ปัญหาที่ต้องระวัง บั๊ก หรือข้อสังเกตเพิ่มเติม
4. **สรุปผลลัพธ์ให้ผู้ใช้:** แจ้งผู้ใช้ว่าได้ทำการสร้าง/อัปเดตไฟล์ `HANDOFF.md` เรียบร้อยแล้ว พร้อมสรุปเนื้อหาสั้นๆ ให้ผู้ใช้ดูในแชท และแนะนำให้เริ่มแชทใหม่แล้วใช้สกิล Resume
