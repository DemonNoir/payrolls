# Handoff Document: Payrolls (ot-calendar)

## Project Overview
โปรเจกต์ Payrolls (ot-calendar) เป็นเว็บแอปพลิเคชัน (PWA) สำหรับจัดการและคำนวณเงินเดือน, โอที (OT), สวัสดิการ, เบี้ยขยัน, และวันหยุด/วันลาต่างๆ ของพนักงาน (เน้นตามกฎหมายและโครงสร้างเงินเดือนไทย) โดยมีการแสดงผลทั้งแบบปฏิทินรายวัน แดชบอร์ดสรุปยอดรอบบิล และกราฟสรุปรายปี

## Current State
ระบบสามารถทำงานได้สมบูรณ์และถูกนำไปใช้งานจริง (Deployed) แล้ว มีการจัดเก็บข้อมูลผ่าน LocalStorage และรองรับการทำงานแบบ Offline ผ่าน Service Worker 
โครงสร้างหลักประกอบด้วย:
- `index.html`: UI หลัก (Calendar, Dashboard, Settings, Annual Summary)
- `js/calc.js`: หัวใจหลักในการคำนวณสูตรรายได้ การหักเงิน และยอดวันหยุดสะสม
- `js/settings.js` / `js/ui.js` / `js/data.js` / `js/app.js`: จัดการข้อมูลและเรนเดอร์หน้าจอ
- `sw.js`: Service Worker จัดการ Caching

## Recent Changes
การเปลี่ยนแปลงล่าสุดที่ได้ทำลงไป:
1. **Service Worker Strategy:** เปลี่ยนกลยุทธ์จาก `Cache-First` เป็น `Network-First` ในไฟล์ `sw.js` เพื่อแก้ปัญหาผู้ใช้ไม่เห็นแอปเวอร์ชันใหม่หลังอัปเดตโค้ด (ตอนนี้ผู้ใช้กดรีเฟรชหน้าเว็บก็จะได้รับโค้ดเวอร์ชันล่าสุดทันที)
2. **Leave System Redesign:** 
   - แยกกระเป๋าวันหยุดสะสมออกเป็น 2 ส่วนชัดเจน: **"สลับหยุด"** และ **"ลาพักร้อน"** 
   - ปรับให้ผู้ใช้สามารถกรอก "ยอดลาพักร้อนสุทธิปัจจุบัน" ลงในหน้าตั้งค่าได้เลย โดยระบบจะคำนวณย้อนกลับ (Reverse-calculate) เพื่อหักลบกับโควต้าอัตโนมัติรายเดือน (4 ชม./เดือน) ให้เอง ทำให้ได้ยอดตรงตามที่กรอกโดยไม่กระทบสูตรสะสมในอนาคต
3. **Diligence & Service Award Logic:** ได้มีการพูดคุยเรื่องการแสดงผลของเบี้ยขยัน (Diligence) และรางวัลอายุงาน (Service Award) ในโหมด "เรียลไทม์" สุดท้ายได้ข้อสรุปว่า **ไม่ให้หารตามวัน (No Prorating)** แต่ให้จ่ายเต็มจำนวนตั้งแต่วันแรกเป็น Lump-sum (All-or-nothing) ตามความตั้งใจของยูสเซอร์

## Open Issues / Notes
- **การแสดงผลโหมดเรียลไทม์:** ในโหมดเรียลไทม์ (ถึงวันนี้) ผู้ใช้อาจตกใจที่ยอดเงินสุทธิกระโดดขึ้นมาสูงมากตั้งแต่วันแรก (เช่น +1000 บาท) ซึ่งเงินส่วนนี้มาจาก "เบี้ยขยัน/รางวัลอายุงาน" ที่ระบบบวกให้เต็มก้อนล่วงหน้า (ตราบใดที่ยังไม่เกิดการลา/ขาดงาน) ยูสเซอร์รับทราบและยืนยันให้คงการทำงานลักษณะนี้ไว้ ห้ามเข้าไปปรับแก้เองโดยไม่ได้รับคำสั่ง
- **การตั้งชื่อและช่องโหว่การสื่อสาร:** ควรระมัดระวังการตั้งค่า PWA Cache Version ในการ Deploy ครั้งถัดๆ ไป แม้จะใช้ Network-First แล้วก็ตาม การอัปเดตไฟล์แบบ Major ยังคงแนะนำให้ขยับเวอร์ชัน `CACHE` ใน `sw.js` เสมอเพื่อเคลียร์ขยะในเครื่องผู้ใช้
