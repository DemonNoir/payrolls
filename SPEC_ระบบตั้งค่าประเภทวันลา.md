# 📝 SPEC: ระบบตั้งค่าประเภทวันลาแบบละเอียด (Leave Type Settings)

> เอกสารนี้สรุปฟังก์ชันการนับ/จัดหมวดวันลาที่ถอดแบบมาจากแอประบบ HR ของบริษัท (ตามภาพหน้าจอที่แนบ) เพื่อส่งต่อให้ AI Agent นำไปพัฒนาเพิ่มในโปรเจกต์ **"ปฏิทิน OT V2"**

---

## 🎯 เป้าหมาย
เพิ่มเมนู **"ตั้งค่าวันหยุดแบบละเอียด"** เข้าไปในแอป โดยให้เข้าถึงผ่านโครงสร้าง **2 ชั้น**:

```
⚙️ Settings (หน้าตั้งค่าหลักของแอป)
   └── 🗂️ ตั้งค่าวันหยุด (เมนูย่อย)
         └── รายการประเภทวันลาแบบละเอียด + ค่าที่ตั้งได้ของแต่ละประเภท
```

- กดจาก Dashboard/เมนูหลัก → เข้า **Settings**
- ใน Settings มีปุ่ม/รายการ **"ตั้งค่าวันหยุด"** → กดเข้าไปเจอหน้าใหม่ที่แสดงรายการประเภทวันลาทั้งหมด (เหมือนภาพตัวอย่าง)
- แต่ละประเภทกดเข้าไปตั้งค่าเชิงลึกได้อีกชั้น (จำนวนโควตา / หน่วยนับ / วันเริ่มนับใหม่ ฯลฯ)

---

## 🗂️ โครงสร้างหมวดหมู่ (อ้างอิงจากภาพ)

แอปต้นแบบแบ่งประเภทวันลาออกเป็น **2 กลุ่มใหญ่**:

### กลุ่ม 1: "ประเภทที่มีข้อจำกัด" (มีโควตา/เพดาน ต้องนับยอดคงเหลือ)
| ชื่อประเภท | หน่วยนับ | ตัวอย่างยอดคงเหลือในภาพ |
|---|---|---|
| วันหยุดพักร้อนประจำปี (Annual Leave) | ชั่วโมง | 4 ชม. |
| ลาป่วยโดยได้รับค่าจ้าง (Paid Sick Leave) | ชั่วโมง | 240 ชม. |
| ลากิจธุระโดยได้รับค่าจ้าง (Paid Personal Leave) | ชั่วโมง | 24 ชม. |
| สลับวันหยุด (Leave Swap) | ชั่วโมง | 0 ชม. |
| ลาเพื่อรับราชการทหาร (Military Service Leave) | วัน | 60 วัน |
| ลาฝึกอบรม (Training Leave) | วัน | 30 วัน |
| ลาเพื่อช่วยเหลือคู่สมรส (Spousal Support Leave) | วัน | 15 วัน |
| ลาเลี้ยงดูบุตร (Parental/Childcare Leave) | วัน | 15 วัน |

### กลุ่ม 2: "ไม่จำกัดประเภทวงเงิน" (ไม่มีเพดาน แค่บันทึกไว้เป็นสถิติ ไม่หักโควตา)
| ชื่อประเภท | หน่วยนับ |
|---|---|
| ลาป่วยโดยไม่ได้รับค่าจ้าง | ชั่วโมง |
| ลาบาดเจ็บจากการทำงาน | วัน |
| ลากิจธุระโดยไม่ได้รับค่าจ้าง | ชั่วโมง |
| ลาแต่งงาน | วัน |
| ลาผ่าตัดทำหมัน | วัน |
| ลางานศพ (ไม่ใช่ญาติสายตรง) | วัน |
| ลางานศพ (ญาติสายตรง) | วัน |
| การพักงาน | วัน |
| การหยุดผลิต | วัน |
| ลาพักฟื้น | ชั่วโมง |

**หลักการสำคัญที่ต้องถอดมาใช้:**
1. **หน่วยนับต่างกันได้ต่อประเภท** — บางประเภทนับเป็น "ชั่วโมง" บางประเภทนับเป็น "วัน" (ไม่บังคับหน่วยเดียวทั้งระบบ)
2. **แยก "มีโควตา" vs "ไม่มีโควตา"** — ประเภทที่มีโควตาต้องมีเลขคงเหลือ (แดง = ใกล้หมด/หมดแล้ว) ส่วนประเภทไม่มีโควตาไม่ต้องแสดงยอดคงเหลือ แค่กดบันทึกการลาได้เลย
3. Label ทุกอันมีคำอธิบายวิธีกรอกกำกับไว้ในวงเล็บ เช่น "(กดชั่วโมงการลา)" หรือ "(กดวันการลา)" — เป็น hint บอกผู้ใช้ว่าประเภทนี้ต้องกรอกหน่วยไหน

## ⚙️ ฟิลด์ข้อมูลที่ควรมีต่อ 1 ประเภทวันลา (Data Model)

```json
{
  "id": "annual_leave",
  "name_th": "วันหยุดพักร้อนประจำปี",
  "unit": "hours",            // "hours" | "days"
  "has_quota": true,           // true = มีเพดาน/ต้องหักยอด, false = ไม่จำกัด
  "quota_total": 0,            // ยอดตั้งต้น (ผู้ใช้กรอกเองได้ตามที่มีอยู่จริง)
  "quota_remaining": 4,        // ยอดคงเหลือ (ระบบคำนวณอัตโนมัติจากการหักลบ)
  "reset_cycle": "yearly",     // ถ้ามี, เช่น รีเซ็ตทุกปี/ทุกเดือน (เผื่ออนาคต)
  "color_tag": "#e05353",      // จุดสีหน้ารายการ ตามภาพตัวอย่าง
  "is_paid": true,             // มีผลต่อการคำนวณรายได้สุทธิหรือไม่
  "visible_in_calendar": true, // แสดงเป็นตัวเลือกในหน้าปฏิทิน/Modal บันทึกวันลาหรือไม่
  "order": 1
}
```

## 📱 UI ที่ต้องสร้าง

### 1) หน้า Settings (เดิม)
- เพิ่มแถวเมนูใหม่: **"🗂️ ตั้งค่าวันหยุด"** พร้อมลูกศร `>` นำไปหน้าถัดไป

### 2) หน้า "ตั้งค่าวันหยุด" (ใหม่ — ชั้นที่ 2)
- แบ่งเป็น 2 section ตามกลุ่มด้านบน (มี header คั่น เช่น "ประเภทที่มีข้อจำกัด" / "ไม่จำกัดประเภทวงเงิน")
- แต่ละแถวแสดง: จุดสี + ชื่อประเภท + หน่วยกำกับในวงเล็บ + ยอดคงเหลือ (ถ้ามีโควตา) + `>`
- กดแต่ละแถว → เปิด modal/หน้าย่อยชั้นที่ 3 สำหรับตั้งค่า:
  - เปิด/ปิดการใช้งานประเภทนี้ (toggle แสดง/ซ่อนจากปฏิทิน)
  - กรอก/แก้ยอดโควตาคงเหลือปัจจุบัน (สำหรับกลุ่มมีโควตา) — ใช้หลัก reverse-calculate แบบเดียวกับที่ทำกับ "ลาพักร้อนสุทธิ" ใน v ปัจจุบันของแอป (ดู ARCHITECTURE.md เดิม)
  - เลือกหน่วยนับ (ชั่วโมง/วัน) — ค่าเริ่มต้นตามตารางด้านบน แต่แก้ไขได้
  - เลือกสี tag

### 3) ผลกระทบต่อ Modal บันทึกวันลาในปฏิทิน
- ตัวเลือกวันลาใน Modal ต้อง **ดึงจากรายการที่ตั้งค่าไว้** (dynamic) แทนการ hardcode แค่ "สลับหยุด" กับ "ลาพักร้อน" แบบในเวอร์ชันปัจจุบัน
- ถ้าประเภทไหนถูกปิดการใช้งานใน Settings จะไม่ขึ้นให้เลือกในปฏิทิน
- ถ้าประเภทมีโควตาและยอดเหลือ = 0 → ควรมี indicator เตือน (เช่น ตัวเลขสีแดง เหมือนภาพต้นแบบ) แต่ไม่บล็อกการกรอก (ให้ผู้ใช้ตัดสินใจเอง)

## ⚠️ ข้อควรระวังสำหรับ AI Agent ที่จะไปทำต่อ

1. **อ่าน `ARCHITECTURE.md` ก่อนแก้ `calc.js`** ตามกฎเดิมของโปรเจกต์เสมอ
2. ระบบวันลาใหม่นี้ต้อง **ไม่ทับ/ไม่พังของเดิม** — "สลับหยุด (OT Swap)" และ "ลาพักร้อน (Annual Leave)" ที่ทำไว้ใน v27 คือ 2 ประเภทแรกของ "กลุ่มมีโควตา" ในสเปกนี้ ให้ mapping เข้าโครงสร้างใหม่โดยไม่เปลี่ยน logic การคำนวณเดิม (โดยเฉพาะ reverse-calculate ของลาพักร้อน)
3. เบี้ยขยัน/รางวัลอายุงาน (lump-sum, no prorating) **ห้ามแตะ** สูตรเดิมเด็ดขาด — เรื่องนี้แยกจากสเปกนี้
4. เพิ่มประเภทวันลาใหม่ทั้งหมดควรทำเป็น **config/array ที่แก้ไขได้** ไม่ hardcode UI แยกทีละประเภท เพื่อให้ในอนาคตเพิ่ม/ลบประเภทได้ง่าย
5. ทุกครั้งที่แก้ JS/CSS ต้อง bump `?v=XX` ใน `index.html` และ `CACHE` ใน `sw.js` ตามกฎเดิมของโปรเจกต์
6. รันเทสต์ `tests/calc-tests.html` ให้ผ่านหลังแก้ไข (และควรเพิ่มเทสต์ใหม่ครอบคลุมประเภทวันลาที่เพิ่มเข้ามา)
7. ดีไซน์ยึดแนว Modern / Glassmorphism / Micro-animation ตามที่ผู้ใช้ชอบ

## 🚀 ขั้นตอนแนะนำสำหรับ Agent ถัดไป
1. เพิ่ม config รายการประเภทวันลา (ตามตารางด้านบน) ในไฟล์ตั้งค่า/state กลาง
2. สร้างหน้า UI "ตั้งค่าวันหยุด" (ชั้น 2) ดึงจาก config นี้
3. เชื่อม Modal บันทึกวันลาในปฏิทินให้ดึงตัวเลือกจาก config เดียวกัน (dynamic แทน hardcode)
4. ย้าย logic ของ "สลับหยุด"/"ลาพักร้อน" เดิมเข้ากรอบ config ใหม่ โดยไม่เปลี่ยนผลลัพธ์การคำนวณ
5. ทดสอบผ่าน `tests/calc-tests.html` ให้ครบ 100%
6. Bump version + อัปเดต `sw.js`
7. Walkthrough และ commit/push ตาม Project Rules
