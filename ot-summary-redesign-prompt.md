# Prompt: Redesign 3 การ์ดข้อมูลด้านล่างการ์ด OT สรุป (การ์ดสีน้ำเงินคงเดิม ไม่ต้องแก้)

## บริบท
แอปคำนวณค่าแรง OT ธีมสีน้ำเงินเข้ม/ส้ม ในหน้า "ปฏิทิน" มีการ์ดสีน้ำเงินเข้มด้านบนแสดงชั่วโมง OT รวมและเงิน OT รวม **(การ์ดนี้ดีอยู่แล้ว ไม่ต้องแก้)**

ที่ต้องแก้คือ **3 การ์ดสีขาวด้านล่าง** ที่ปัจจุบันเรียงแนวนอนเท่ากันหมด:
1. วันหยุดสะสม
2. รายการหัก + สุทธิ (อยู่การ์ดเดียวกัน)
3. เงินออกอีกกี่วัน

### ปัญหาของเดิม
- การ์ดที่ 2 มี 2 ตัวเลขคนละความหมายอยู่ในพื้นที่เดียวกัน (ยอดหัก vs ยอดสุทธิที่ได้รับจริง) โดยไม่มี hierarchy หรือเส้นแบ่งชัดเจน เสี่ยงอ่านสลับ
- ทั้ง 3 การ์ดมีน้ำหนักภาพเท่ากันหมด ทั้งที่ "สุทธิที่ได้รับ" ควรเด่นที่สุดเพราะเป็นตัวเลขที่ผู้ใช้แคร์มากที่สุด

## ดีไซน์ใหม่ (ยืนยันแล้ว ให้ทำตามนี้)

### Layout
- แถวบน: **2 การ์ดเรียงคู่กัน** (grid 2 คอลัมน์เท่ากัน)
  - การ์ดซ้าย: วันหยุดสะสม
  - การ์ดขวา: เงินออกอีกกี่วัน
- แถวล่าง: **1 การ์ดเต็มความกว้าง** สำหรับ รายการหัก + สุทธิ

### การ์ดวันหยุดสะสม
- ไอคอน calendar-time ด้านบน
- label "วันหยุดสะสม" ตัวเล็กสีรอง (secondary text)
- ตัวเลข "0 ชม." ตัวใหญ่กว่า label ชัดเจน (ประมาณ 20px, น้ำหนัก 500)

### การ์ดเงินออกอีก
- ไอคอน wallet ด้านบน
- label "เงินออกอีก" ตัวเล็กสีรอง
- ตัวเลข "22 วัน" ตัวใหญ่ (20px, น้ำหนัก 500)
- เพิ่มบรรทัดวันที่กำกับด้านล่างตัวเล็กมาก สีจาง เช่น "17 ก.ค. 2569" (แก้ปัญหาที่ของเดิมไม่มีวันที่ชัดเจนกำกับ)

### การ์ดรายการหัก + สุทธิ (จุดสำคัญที่สุด)
โครงสร้างภายใน 1 การ์ด แบ่งเป็น 2 โซนด้วยเส้นคั่น (divider):

**โซนบน (รายการหัก):**
- แถวเดียว จัดชิดซ้าย-ขวา: label "รายการหัก" (มีไอคอน trending-down เล็กๆ นำหน้า สีแดง/ส้ม) ทางซ้าย, ตัวเลข "-1,090.00 บาท" ทางขวา
- ตัวเลขนี้ **สีแดง/ส้ม (danger)** ขนาดกลาง (~14px) น้ำหนัก 500 — เล็กกว่ายอดสุทธิชัดเจน เพื่อบอกว่านี่เป็นแค่รายการหัก ไม่ใช่ตัวเลขหลัก

**เส้นคั่น** บางๆ (1px, สีเส้นแบบ border ปกติของธีม)

**โซนล่าง (สุทธิ):**
- แถวเดียว จัดชิดซ้าย-ขวา แนว baseline ด้านล่าง: label "สุทธิที่ได้รับ" ทางซ้าย (สีข้อความหลัก), ตัวเลข "13,162.67 บาท" ทางขวา
- ตัวเลขนี้คือจุดเด่นที่สุดของการ์ดทั้งหมด: **สีเขียว (success)**, ขนาดใหญ่สุด ใช้ `clamp()` ปรับตามความกว้างจอ (เช่น `clamp(20px, 7vw, 26px)`) เพื่อไม่ให้ล้นขอบการ์ดบนจอมือถือทุกขนาด, น้ำหนัก 500
- หน่วย "บาท" ต่อท้ายตัวเลขให้เล็กกว่าตัวเลขหลักและสีรอง

## ข้อกำหนดทั่วไป (ยังต้องคงไว้)
- **Responsive**: ทดสอบ breakpoint 320px–430px ตัวเลขต้องไม่มีวันล้นขอบการ์ด ใช้ `clamp()` หรือ flex-shrink
- **สีความหมาย (semantic color)**: รายการหัก = สีแดง/ส้ม (danger), สุทธิ = สีเขียว (success) — ใช้ CSS variable ของธีม เช่น `--text-danger`, `--text-success` ไม่ hardcode สี เพื่อให้รองรับ dark mode อัตโนมัติ
- **Typography**: ตัวเลขทั้งหมดใช้ tabular-nums (font-variant-numeric: tabular-nums) เพื่อให้ตัวเลขเรียงสวย ไม่กระตุก
- **Card style**: การ์ดขาว/พื้นผิว ใช้ radius มุมโค้ง ~12px, border บางๆ 0.5-1px สีตามธีม, padding สม่ำเสมอทุกการ์ด (~1rem)
- **Icon**: ใช้ icon set ที่มีอยู่ในโปรเจกต์ (หรือ outline icon library เช่น Tabler icons) — calendar-time, wallet, trending-down
- **Dark mode**: ทุกสีต้องอ่านออกได้ทั้ง light/dark mode ผ่าน CSS variable ของธีม ห้าม hardcode hex
- **การ์ดสีน้ำเงินเข้มด้านบน (OT รวม/เงิน OT รวม) ไม่ต้องแก้ ให้คงเดิมทุกประการ**

## สิ่งที่อยากให้ Agent ส่งมอบ
1. Component (ตามสแต็กของโปรเจกต์ เช่น React/HTML/CSS) ของ 3 การ์ดด้านล่างตามสเปกและ layout ด้านบน
2. Mock data props ที่รับค่า: `accumulatedLeaveHours`, `daysUntilPayday`, `paydayDate`, `deductionAmount`, `netAmount`
3. ทดสอบ responsive ที่ 320px, 375px, 430px ให้ตัวเลขไม่ล้นขอบ
4. รองรับ light/dark theme ผ่าน CSS variable ที่มีอยู่แล้วในโปรเจกต์
5. อธิบายสั้นๆ ว่าจัดการปัญหาตัวเลข 2 ความหมายปนกันในการ์ดเดิม (หัก vs สุทธิ) อย่างไรด้วย hierarchy และเส้นคั่น
